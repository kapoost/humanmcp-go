A. Subdomena u bliskiego (dla rodziny)

Ty (host) stawiasz serwer bliskiej osoby na swoim koncie Fly. Ona nie potrzebuje konta, karty, ani terminala.

bliski podaje imie i bio
→
host odpala komende
→
serwer dziala
→
host wysyla link + token
fly apps create alice-humanmcp --org personal fly volumes create humanmcp_data --size 1 --region ams --app alice-humanmcp -y fly secrets set EDIT_TOKEN="$(openssl rand -hex 16)" \ AUTHOR_NAME="Alice" \ DOMAIN="alice-humanmcp.fly.dev" \ --app alice-humanmcp fly deploy --app alice-humanmcp
Host placi 0 zl (free tier: 3 maszyny na konto).
Wiecej niz 3 osoby → $1.94/msc za kazda kolejna VM (256MB).
Bliski loguje sie tokenem, dodaje tresc, nie potrzebuje terminala.
Opcjonalnie: subdomena alice.twojadom.net → fly certs add alice.twojadom.net --app alice-humanmcp

Co bliski dostaje:

Serwer: https://alice-humanmcp.fly.dev Login: https://alice-humanmcp.fly.dev/login Follow: https://humanmcp.net/humannetwork.html?add=https://alice-humanmcp.fly.dev Connect: https://alice-humanmcp.fly.dev/connect

B. Fly.io (wlasny deploy)

zainstaluj flyctl
→
fly auth signup (email + karta)
→
bash setup.sh
→
gotowe
git clone https://github.com/kapoost/humanmcp-go.git cd humanmcp-go bash setup.sh
Skrypt pyta o imie, bio, region. Tworzy app, volume, klucze, deploy.
Karta kredytowa wymagana przez Fly do weryfikacji (anty-abuse), nie jest obciazana.
Free tier: 3 shared VM (256MB), 3GB persistent volume, HTTPS, auto-sleep.

C. VPS (Docker)

ssh root@twoj-vps git clone https://github.com/kapoost/humanmcp-go.git cd humanmcp-go docker build -t humanmcp . docker run -d --name humanmcp \ -p 8080:8080 \ -v humanmcp_data:/data \ -e EDIT_TOKEN=secret \ -e AUTHOR_NAME=yourname \ -e DOMAIN=yourdomain.com \ humanmcp
Potrzebujesz nginx jako reverse proxy + certbot (Let's Encrypt) dla HTTPS.
Hetzner CX22: ~16 zl/msc, DigitalOcean: ~$4/msc, OVH: ~$3.50/msc.

D. Raspberry Pi

git clone https://github.com/kapoost/humanmcp-go.git cd humanmcp-go go build -o humanmcp ./cmd/server/ EDIT_TOKEN=secret AUTHOR_NAME=yourname ./humanmcp
Dziala na RPi 3/4/5, NAS (Synology z Docker), starym laptopie.
Tunel do internetu: Cloudflare Tunnel (darmowy) lub Tailscale Funnel.
Binary: ~7MB, zuzwa ~15MB RAM. Zero zewnetrznych zaleznosci.
